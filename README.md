# diary
# そだてノート 仕様書

日記 × タスク × 習慣 を掛け合わせたペット育成PWA。
単一HTMLファイル(`index.html`)で完結し、GitHub Pagesにそのままデプロイ可能。

---

## 1. コンセプト

- プレイヤー(自分自身の分身)は日記・タスク・習慣の実績を積むとレベルアップする
- ペットは**1匹だけ**を長く大事に育てる方式(複数飼育はしない)
- ペットの見た目は **rough.js** でその場生成する手描き風イラスト(素材の画像ファイルは使わない)
- 進化は12段階、姿形が大きく変わっていく「じっくり系」
- 最終形態到達後も終わりにせず、記念日ごとに特別演出が入り続ける(D案)

---

## 2. 画面構成

| 画面 | 内容 |
|---|---|
| ホーム | ペット表示、げんき/なかよしバー、今日のタスク・習慣一覧 |
| 日記 | テキスト入力・保存、過去ログ一覧 |
| タスク | 追加・完了チェック・削除 |
| 習慣 | 追加・今日の達成チェック・streak表示・削除 |
| ペット詳細 | 名前編集、進化チャート(12段階)、記念日/実績一覧 |

画面下部のタブバーで5画面を切り替える(SPA、ページ遷移なし)。

## 3. ゲームデザイン

### 3-1. プレイヤー(レベル)
- 経験値(EXP)を貯めるとレベルアップ
- 必要EXP = `100 + (レベル-1) × 40`
- EXP獲得元: 日記記録・タスク完了・習慣達成

### 3-2. ペット

#### ステータス
| ステータス | 説明 | 増減 |
|---|---|---|
| げんき (hunger) | 0〜100。**ゆるめ設定**、放置しても致命傷にはならない(下限20でストップ) | タスク完了・日記記録で回復。未活動日数×4/日で自然減少 |
| なかよし (affection) | 0〜100 | 日記記録・習慣達成で上昇 |
| 進化ポイント (evoPoints) | 進化段階を決める累積値 | 日記+6、タスク完了+5、習慣達成+3〜(streak/5切り捨て) |

#### 進化段階(全12段階・しきい値)
| # | 名称 | 必要ポイント | 見た目の変化 |
|---|---|---|---|
| 1 | たまご | 0 | 卵のみ |
| 2 | ひび割れたたまご | 20 | ひびと覗く顔 |
| 3 | うまれたて | 45 | 体・耳が生える |
| 4 | よちよち期 | 80 | 体が少し大きく |
| 5 | 幼体 | 125 | さらに成長 |
| 6 | 少年期 | 180 | 角が生える |
| 7 | 成長期 | 245 | 模様が出る |
| 8 | 若獣期 | 320 | 体格アップ |
| 9 | 成獣期 | 410 | 翼が生える |
| 10 | 覚醒期 | 520 | 体色変化 |
| 11 | 完成形 | 650 | オーラの輪(1本) |
| 12 | 真化 | 800 | オーラの輪(複数)・最終フォルム |

#### 完成形到達後(D案)
- 姿の基本形は12段階目で固定
- 記念日(下記)を重ねるたびにオーラの輪が増える
- 「〇〇と暮らして n日目」のカウントは無制限に継続表示

#### 記念日(マイルストーン)
`7 / 14 / 30 / 50 / 100 / 200 / 300 / 500 / 750 / 1000 / 1500 / 2000` 日を誕生日(初回起動日)から起算し、到達するとトースト通知+実績バッジとして記録される。

---

### 3-3. 日記
- 保存すると **EXP = 10 + min(20, 文字数/10切り捨て) + min(15, 連続記録日数)**
- 保存のたびに: 進化ポイント+6、なかよし+8、げんき+5
- **自動下書き機能**: その日タスク/習慣を達成していると、日記タブを開いた時に「今日は『○○』を達成した!」という一文が自動でテキストエリアの先頭に挿入される(1日1回、未編集分のみ)

### 3-4. タスク
- テキスト入力で追加、チェックで完了
- 完了時: EXP+8、進化ポイント+5、げんき+12(=エサをあげる演出)
- チェック解除で取り消し可能、削除ボタンあり

### 3-5. 習慣
- テキスト入力で追加、「今日」ボタンで当日分を達成
- 連続達成日数(streak)を記録。前日達成していれば+1、途切れていたら1にリセット
- 達成時: EXP = 6 + streak/3切り捨て、進化ポイント = 3 + streak/5切り捨て、なかよし+4
- もう一度押すと当日分を取り消し可能

```json
    "lastFedDate": null,
    "milestonesHit": []
  },

    { "id": "string", "date": "YYYY-MM-DD", "text": "string", "exp": 0, "auto": false }
  ],
  "tasks": [
    { "id": "string", "text": "string", "done": false, "doneDate": null }

  "draftedFor": null

```

localStorageが使えない環境ではメモリ上の変数にフォールバックする(ページ再読み込みで消える点に注意)。

## 5. 技術構成

- **形式**: 単一HTMLファイル(`index.html`)。CSS・JSともにインライン
- **フォント**: Shippori Mincho(見出し)/ Zen Kaku Gothic New(本文)/ JetBrains Mono(数値)※Google Fonts CDN
- **ペット描画**: rough.js 2.1.1(cdnjs)による手描き風canvas描画。段階ごとにパーツ(耳・角・模様・翼・オーラ)を出し分け
- **PWA対応**: `manifest.json`と簡易Service Workerをどちらも実行時にBlob生成してインライン埋め込み(別ファイル不要)
- **保存先**: ブラウザのlocalStorage(将来的にSupabase連携も拡張可能)
- **デプロイ**: GitHub Pagesにそのままpush(このリポジトリ構成と同じ運用を想定)
